# 機能要件

## プロジェクト概要

個人の収支を管理するWebアプリケーション。
固定費・変動費を区別して支出を記録し、後々は収入も併せて管理できるよう拡張可能な設計にする。

本プロジェクトは案件で使用するVue.js / Nuxt / NestJSの学習を目的とする。

## ユーザー

- 単一ユーザー前提（マルチテナント対応は将来検討）
- 初期実装では仮ユーザー1名をseedで投入し、ログイン済みとして扱う
- 認証機能は後続フェーズでBetter Auth等を用いて追加予定

## スコープ

### 今回のスコープ（MVP）

- 取引（Transaction）のCRUD
  - 金額・日付・カテゴリ・メモ・固定/変動フラグを登録
  - 一覧表示・詳細表示・編集・削除
- カテゴリマスタの参照
  - seedで一般的な収支カテゴリを投入

### 将来スコープ（今回は実装しない）

- 収入の登録・管理（DBカラムだけ用意して当面は支出固定）
- 認証（ログイン・サインアップ）
- 月次集計・グラフ表示
- カテゴリのユーザーカスタマイズ
- 予算設定・通知

## 機能詳細

### 取引（Transaction）

| 項目 | 説明 |
|---|---|
| 金額 | 正の整数で保持 |
| 日付 | 取引が発生した日付 |
| カテゴリ | カテゴリマスタから選択 |
| メモ | 自由記述（任意） |
| 固定/変動 | 固定費か変動費かを示すフラグ |
| 種別 | 収支区分（収入 or 支出）。当面は支出固定 |

### カテゴリ

| 項目 | 説明 |
|---|---|
| 名前 | カテゴリ名（食費、交通費など） |
| 種別 | 収入用 / 支出用 |

#### 初期カテゴリ（seed）

- 支出: 食費 / 交通費 / 住居費 / 光熱費 / 通信費 / 娯楽費 / 医療費 / 衣服費 / その他
- 収入: 給与 / 賞与 / 副業 / その他

## 画面構成（フロントエンド）

| 画面/UI | 役割 |
|---|---|
| 一覧画面 | 取引の一覧表示。日付の新しい順に並べる |
| 登録モーダル | 一覧画面から開く新規登録フォーム |
| 編集モーダル | 一覧画面から開く既存取引の編集フォーム |

削除は一覧画面の各行または編集モーダル内から行う。
